There is 
	TexLive
	TexLive-LaTex
	TexLive-Xetex
	TexLive-LuaTex

LaTeX, XeTex, and LuaTex are the recipes. The set of instructions and markup that defines the structure and ingredients of your final document. TeX Live is the kitchen, stocked with all the necessary tools and supplies—the oven, pans, utensils, and ingredients—that you need to follow the recipe and produce the finished dish. You need both a recipe (LaTeX, XeTex, LuaTex) and a kitchen (TeX Live) to create a beautifully typeset document. 


There is 
	BibLaTex
	Biber

Biber is the backend program that processes citation data for the
BibLaTeX package, while BibLaTeX is the LaTeX package that provides a flexible and powerful interface for managing and formatting bibliographies and citations. In essence, BibLaTeX is the user-facing LaTeX tool, and Biber is the separate program it calls upon to handle the actual sorting, formatting, and data processing of bibliography information, offering more advanced features like full Unicode support and customizability than the older BibTeX

	*TeX uses the kpathsea library for finding files
		-- the utility for listing kpathsea search paths is kpsepath
		-- kpsepath calls kpsewhich via the script kpsetool. 
		-- kpsewhich searches paths for 
			==> .tex files
			==> .cls files
			==> .sty files
			==> all types of font files 

	*kpsewhich identifies three different types of paths
		-- Paths ending with a double slash are to be searched recursively. 
		-- Only the database (and not the entire disk) will be searched if paths are prepended by two 
		exclamation marks. In TeX Live, most of the trees are specified with �!!�
		-- Paths without either double slashes or exclamation marks are searched without recursion.

	*Kpathsea looks for ls-R files along the TEXMFDBS path. It is best for this to contain all and 
	only those hierarchies from $TEXMF which are specified with !!�and also to specify them with !! 
	in TEXMFDBS.
	
	*ls-R is a .txt file (/usr/local/share/texmf/ls-R)The recommended way to create and maintain �ls-
	R� is to run $ mktexlsr
