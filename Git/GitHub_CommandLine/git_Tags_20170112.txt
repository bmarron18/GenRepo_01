##################
The Skinny on Tags
#################

	a. two kinds (annotated and lightweight) but only annotated worth using 
	b. Tags are tied to commits!
	c. Tags can only be manipulated by cloning a repo locally then pushing back up to the remote repo
	d. GitHub can only show Tags
	e. clicking on a Tag in GitHub brings up the repo AS IT WAS at the time of the commit tied to the Tag 
	f. common version coding for tags includes
		-a	#alpha
		-b	#beta
		-rc	#release candidate
	g. to deal effectively with Tags use a command-line interface

##############################################
Helpful Commands for Tags in Git
(assumes have cloned a local copy of the repo)
###############################################
http://stackoverflow.com/questions/1028649/how-do-you-rename-a-git-tag

$ git tag					#list all tags assoc w/ the repo
$ git tag -l "v3.1*"				#list the subset of tags beginning w/ "v3.1"
$ git show v3.1-rc2				#output the details of tag v3.1-rc2

$ git log --pretty=oneline			#list all commits for the repo as one-liners
		

$ git tag new old				#build an alias of the old tag name
$ git tag -d old				#delete old tag
$ git push origin :refs/tags/old		#delete the tag on your remote location(s):
$ git push origin --tags			#add new tag to the remote location
$ git pull --prune --tags			#make sure that the other users remove the deleted tag (if working directly fr same repo)
